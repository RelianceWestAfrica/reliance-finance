# Guide AG (Administrateur General)

> L'**AG** est le validateur ultime pour les dossiers depassant le seuil
> Groupe. Vous validez les engagements les plus significatifs. Vue
> agregee Groupe + acces a tous les SPV / Filiales.

## Mission

- Valider les FDA / FD / BC depassant le seuil AG (souvent les
  investissements CAPEX strategiques)
- Examiner les anomalies CRITICAL escaladees par DFG ou Controleur
  Interne
- Suivre les indicateurs strategiques Groupe (cash, dossiers, urgences)
- Decider des arbitrages cash entre filiales / SPV

## Vue strategique

Tableau de bord AG accessible des le login. Inclut :

- **Cash position Groupe** : aggregation toutes filiales en XOF + EUR
  (taux du jour seede)
- **Cash forecast Groupe sur 13 semaines** + alertes rupture
- **Dossiers en attente de votre signature** : peut etre 0 la plupart
  des jours
- **Volume engage** par filiale / projet sur le trimestre
- **Top 5 anomalies CRITICAL** non resolues

## Workflow de validation AG

**Demandes de fonds > Filtre "En attente AG"**

Generalement < 5 dossiers / mois. Pour chaque dossier :

1. Verifier l'**historique complet de validation** :
   - Validation N1 (DAF Pays) : qui, quand, commentaire ?
   - Validation N2 (Tresorier / Finance Groupe) : qui, quand, commentaire
     ?
   - Validation Groupe (DFG) : qui, quand, commentaire ?
2. Verifier la **coherence strategique** :
   - Le dossier est-il aligne avec les orientations Groupe ?
   - Le ROI / impact metier est-il documente ?
   - L'impact cash est-il acceptable (cf. cash forecast) ?
3. Bouton **Valider AG** (commentaire optionnel mais recommande) ou
   **Rejeter** (motif obligatoire)
4. Votre signature cryptographique est ajoutee, c'est la **derniere etape**

Apres validation AG, le dossier devient une autorisation d'engagement :
- BC peut etre genere
- Paiement peut etre prepare (apres reception PV + facture)

## Validation des changements de RIB strategiques

Si un fournisseur **SENSIBLE** (= sensibilite SENSITIVE) change son RIB,
les anomalies generees peuvent vous etre escaladees automatiquement.

Procedure :

1. Examiner l'historique des RIBs (Fournisseurs > Fiche > Historique RIB)
2. Verifier les validations DAF + DFG en N1/N2
3. Consulter le Controleur Interne pour son analyse
4. **Approuver** ou **Bloquer** definitivement la suite du processus

Si bloque : tous les paiements vers ce fournisseur sont suspendus jusqu'a
votre nouvelle decision.

## Arbitrage cash entre filiales

Si une filiale est en risque de rupture (alerte du cron quotidien
06h30), vous pouvez :

**Cash forecast > Vue Groupe > Arbitrer**

- Visualiser la position cash de chaque filiale
- Proposer un transfert intergroupe : Filiale Togo prete a SPV Lome
- Le transfert genere :
  - Un BC interne (entre entites)
  - Une journalisation comptable inter-compagnie
  - Pas de paiement bancaire reel mais un mouvement comptable

Ces arbitrages sont **tracables** dans l'audit log avec votre signature.

## Acces special : tout le Groupe

En tant qu'AG, vous avez visibilite **integrale** sur :

- Toutes les Filiales, SPV, et leurs Projets
- L'audit log integral
- Toutes les anomalies (peu importe l'assignataire)
- Tous les fournisseurs (peu importe l'entite)

Vos consultations sont **journalisees** comme tout autre acteur, dans
la chaine cryptographique.

## Procedure exceptionnelle : derogation

Dans certains cas exceptionnels (force majeure, contraintes
operationnelles), un dossier peut etre execute **sans suivre l'integralite
du workflow normal**. Cette procedure exceptionnelle requiert :

1. Validation manuelle de **vous-meme** + Conseil d'Administration
2. Justification ecrite obligatoire (motif force majeure + impact si
   non execution)
3. Saisie a posteriori du dossier dans Reliance Finance avec mention
   "DEROGATION" dans le commentaire
4. Audit specifique par le Controleur Interne dans les 48h

Ces derogations sont visibles dans un **rapport mensuel** specifique
soumis au Conseil d'Administration.

## En cas d'absence prolongee

Designez un **suppleant** par decision du Conseil d'Administration. Votre
DSI / Admin systeme creera son membership avec role AG sur le Groupe.

A votre retour, vous pouvez :

- Voir toutes les decisions prises par votre suppleant (audit log filtre
  actor = suppleant)
- Valider / contester via un rapport au CA
- Reactiver vos propres droits en revokant ceux du suppleant
